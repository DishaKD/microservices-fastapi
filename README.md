# Microservices with FastAPI

This project is a modular microservices architecture built using Python and FastAPI. It consists of multiple independent services and an API Gateway that routes traffic securely.

## System Architecture

The project contains the following components:

- **API Gateway (Port 8000):** The main entry point. Handles JWT authentication, request logging, structured error handling, and routes external requests to internal microservices. It also hosts the unified Swagger UI documentation.
- **Student Service (Port 8001):** Manages student records.
- **Course Service (Port 8002):** Manages course records.
- **Auth Service (Port 8003):** Handles user registration, secure password hashing (using `bcrypt`), and JWT token generation.

## Prerequisites

- **Python 3.12+** (Developed and tested on Python 3.14)
- **pip** (Python package installer)

---

## 🚀 Environment Setup

1. **Create a Virtual Environment:**
   Run the following command in the root directory to create an isolated Python environment.
   ```bash
   python -m venv venv
   ```

2. **Activate the Virtual Environment:**
   - **On macOS/Linux:**
     ```bash
     source venv/bin/activate
     ```
   - **On Windows:**
     ```cmd
     venv\Scripts\activate
     ```

3. **Install Dependencies:**
   Install all required packages from `requirements.txt`.
   ```bash
   pip install -r requirements.txt
   ```

*(Note: If you run into build errors regarding `pydantic-core` on newer Python versions, ensure you are using the virtual environment and have the correct Rust compiler installed if required by your OS, or pull the pre-compiled wheels if available).*

---

## ▶️ Running the Microservices

You must run each microservice and the API gateway simultaneously. Open **4 separate terminal tabs**. 

Make sure your virtual environment is **activated in every tab**!

**Tab 1 - API Gateway:**
```bash
cd gateway
uvicorn main:app --reload --port 8000
```

**Tab 2 - Student Service:**
```bash
cd student-service
uvicorn main:app --reload --port 8001
```

**Tab 3 - Course Service:**
```bash
cd course-service
uvicorn main:app --reload --port 8002
```

**Tab 4 - Auth Service:**
```bash
cd auth-service
uvicorn main:app --reload --port 8003
```

## 📖 API Documentation & Testing

Once all the services are running, you can interact with the entire system through the API Gateway's unified Swagger UI:

👉 **http://localhost:8000/docs**

### How to Authenticate:
The API Gateway secures the `student` and `course` endpoints using JWT Bearer tokens. 

1. Go to the Swagger UI (`/docs`).
2. Scroll down to `POST /gateway/auth/register` and create a new user account via the JSON body.
3. Scroll back to the top of the page and click the green **Authorize** padlock button.
4. Enter the credentials you just registered.
5. You can now execute requests against the secured `/gateway/students` and `/gateway/courses` endpoints!

## 🛠️ Key Features Added

* **JWT Authentication:** Tokens are generated by the Auth Service and validated at the API Gateway layer.
* **Global Request Logging:** A middleware monitors and records exact process times and HTTP methods for every request.
* **Unified Error Handling:** Any 4xx or 5xx error (whether from the gateway or a downstream service) is caught and formatted into a highly readable JSON schema.
